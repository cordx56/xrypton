# crypton

E2E暗号化されたチャットサービス。

## 方針

- すべてのメッセージ、データはPGPで暗号化され、armored形式のテキストファイルで保存される。
- すべてWeb技術で実装される。
- 暗号、署名鍵の生成はwasmを用いてクライアントサイドで行われる。

## 実装

- crypton-commonにPGP実装の共通部分を持つ。
- crypton-wasmにPGP実装をフロントエンドで利用できるようにするためのwasm実装を行う。
- crypton-apiにはサーバのAPI実装を行う。
- crypton-webにはwebフロントエンドを実装する。

## 実装上の注意

以下を必ず守れ。

- 通信のスキーマを十分に確認する。
- ライブラリをよく調べ、最新のものを用いる。
- E2E暗号化の観点に反する実装は行わない。
- 定期的にGitコミットを残し、作業ログを保持する。
