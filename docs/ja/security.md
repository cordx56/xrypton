# セキュリティ

Xryptonはセキュリティを重視しています。

## 署名と暗号化

Xrypton上のほとんどのデータは署名され、メッセージとして送られたデータは暗号化されます。
以下にどのようなデータが署名または暗号化されるかを示した表を掲載します。

| データ             | セキュリティ  |
|--------------------|---------------|
| プロフィール画像   | 署名          |
| 名前               | 署名          |
| ステータス         | 署名          |
| 自己紹介           | 署名          |
| チャンネル名       | なし          |
| スレッド名         | なし          |
| 更新日時           | なし          |
| メッセージ         | 署名 / 暗号化 |
| 添付ファイル       | 署名 / 暗号化 |
| ファイルメタデータ | 署名 / 暗号化 |

## 情報が保管される場所

情報の保管場所は次のようになっています。

| データ               | 場所                |
|----------------------|---------------------|
| PGP 公開鍵           | サーバ / ユーザ端末 |
| PGP 秘密鍵           | ユーザ端末          |
| プロフィール情報     | サーバ              |
| 暗号化済みメッセージ | サーバ              |
| 暗号化済みファイル   | サーバ              |
| AT Protocol 認証情報 | ユーザ端末          |

## リアルタイムセッション

極めて高いセキュリティを要求されるコミュニケーションにはリアルタイムセッションが利用できます。

リアルタイムセッションはユーザが暗号化された接続経路情報を受け渡すことで、それぞれが直接通信をできるようにします。
通信はサーバを介さないため、サーバには暗号化されたメッセージすら残りません。

リアルタイムセッションでは、参加するユーザがそのセッション限りのPGP鍵を生成して、メッセージを暗号化します。
この鍵はブラウザのタブを閉じたり、セッションを退出すると端末から削除され、受信したメッセージは一切読めなくなります。
